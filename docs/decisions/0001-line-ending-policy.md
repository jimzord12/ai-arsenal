# ADR 0001: Repository line-ending policy

## Status

Accepted on 2026-07-12.

## Context

Phase 8 clean-checkout validation on Windows exposed that `core.autocrlf=true` checked tracked LF files out as CRLF when the repository had no `.gitattributes` policy. That made `prettier --check .` fail and changed the byte-sensitive `features-cli` help fixture line endings in a fresh clone.

## Decision

The repository uses `.gitattributes` with `* text=auto eol=lf` so tracked text files check out with LF line endings on Windows and Linux.

## Consequences

Formatter checks, workflow files, TypeScript sources, Markdown docs, and test fixtures have stable line endings across clean checkouts. Windows command shims generated by package managers remain generated consumer artifacts and are not tracked.
